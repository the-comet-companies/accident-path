# Content Plan

## Content Strategy Overview

Build topical depth through content clusters. Each cluster has a pillar page, spoke pages, and supporting tools/guides. All content must be original, substantial, and people-first.

**CRITICAL:** Every page must have an assigned target keyword from `docs/seo/KEYWORD-MAP.md` before content writing begins. See `SEO-STRATEGY.md` Section 0 for the keyword map format.

### Minimum Content Depth Requirements

| Page Type | Min. Words | Differentiators Required (pick 2+) |
|-----------|----------:|------------------------------------|
| Accident hub | 2,500 | Embedded tool, attorney review badge, video, PDF download |
| Guide page | 1,500 | FAQ content block (PAA-optimized), checklist, state-specific section |
| Tool page | 800 | Interactive tool (primary), supporting educational content |
| City page | 1,200 | Local data (hospitals, courts, roads), accident stats, local attorneys |
| State page | 2,000 | Law overview, deadlines table, state-specific tools, attorney review |
| Blog post | 1,000 | Timeliness, original angle, internal links to hub pages |
| Comparison page | 1,500 | Side-by-side table, pros/cons, decision framework |

## 1. Accident Type Content Hubs

Each hub is a pillar page at `/accidents/{type}` with the following content modules:

### Hub Template (apply to all 15 types)

| Section | Content |
|---------|---------|
| Overview | What this accident type is, how common, who's affected |
| Common Causes | 5-10 causes with explanations |
| Likely Injuries | Link to relevant injury pages |
| Immediate Steps | Numbered action list (first 15 min, 24 hrs, 7 days) |
| Evidence Checklist | Interactive checklist component |
| Timeline Risks | Statute of limitations, insurance deadlines |
| Insurance Issues | Claim process, common insurer tactics |
| When You Need a Lawyer | Clear guidance (not advice) |
| State-Specific Notes | Dynamic content by jurisdiction |

### The 15 Accident Type Hubs

1. **Car Accidents** — The anchor hub. Most traffic potential.
2. **Truck Accidents** — Commercial vehicle complexity, federal regulations
3. **Motorcycle Accidents** — Unique injury patterns, bias issues
4. **Uber/Lyft Accidents** — Rideshare insurance layers, multiple parties
5. **Pedestrian Accidents** — Severity of injuries, right-of-way laws
6. **Bicycle Accidents** — Urban focus, helmet laws, road sharing
7. **Slip and Fall** — Premises liability, documentation importance
8. **Dog Bites** — Strict liability states, homeowner insurance
9. **Construction Injuries** — Worker's comp vs. third-party claims
10. **Workplace Injuries** — Worker's comp process, employer liability
11. **Wrongful Death** — Family members' rights, estate claims
12. **Premises Liability** — Property owner responsibility
13. **Product Liability** — Defective products, manufacturer liability
14. **Traumatic Brain Injury** — Long-term impact, diagnosis challenges
15. **Spinal Injuries** — Catastrophic injury, lifetime costs

## 5. Content Production Workflow

### Phase 1: Cornerstone Content (Days 16-35)

Publish 25+ cornerstone pages:

- Home page
- 5 highest-volume accident type hubs (car, truck, motorcycle, slip/fall, workplace)
- 5 core guide pages (including "Am I at fault?" and "Settlement vs. Lawsuit")
- 5 tool pages + Statute of Limitations Countdown
- 3 state pages (CA, AZ)
- About / How It Works
- Contact / Intake
- Privacy + Terms + Disclaimers + CCPA "Do Not Sell" page
- **5-10 blog posts targeting long-tail keywords from Reddit harvester**
- **2-3 downloadable PDF lead magnets** (post-accident checklist, insurance call script, evidence guide)

### Phase 2: Expansion (Days 36-60)

- Remaining 10 accident type hubs
- 5 additional guide pages
- Injury type pages
- State-specific guides reviewed by counsel
- **16 city pages with unique local data**
- **Spanish Tier 1: Bilingual UI + intake wizard**
- **8-12 additional blog posts**

### Phase 3: Depth (Days 61-90)

- Additional spoke content for each cluster
- **Sustained blog publishing: 3 posts/week (Reddit harvester + news)**
- Linkable assets (infographics, data studies)
- Downloadable PDF resources
- **Spanish content: translate top 5 accident hubs**
- **Video content: 60-90 second explainers for top 5 hubs**
- **Comparison page series** (settlement vs trial, PI vs workers comp, etc.)

## 6. Content Quality Requirements

### Every Page Must Have
- [ ] Unique title tag and meta description
- [ ] One H1, semantic heading hierarchy
- [ ] At least 3 internal links to related content
- [ ] CTA to intake/matching flow
- [ ] Appropriate disclaimers
- [ ] Review badge (attorney or medical reviewer)
- [ ] Structured data (breadcrumb + page-specific schema)
- [ ] Mobile-optimized layout
- [ ] Original, substantial content (not thin rewrites)

### Review Requirements
| Content Type | Reviewer |
|-------------|----------|
| Legal information | Licensed attorney in relevant state |
| Medical content | Medical professional |
| General educational | Editorial review |
| Tool outputs | Compliance + editorial review |

### AI Content Policy
- AI can draft content
- All drafts MUST be reviewed by qualified humans
- Enhanced with original insights and real-world knowledge
- Fact-checked against current laws
- Never published without human sign-off
