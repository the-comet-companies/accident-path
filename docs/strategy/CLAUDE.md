## RULES

### ALWAYS
1. Read `COMPLIANCE.md` before writing ANY user-facing copy
2. Use careful legal language — never imply legal advice
3. Include disclaimers on every page
4. Use Tailwind CSS only
5. JSON CMS for all content (never hardcode)
6. Target 90+ Lighthouse scores
7. Mobile-first, accessible (WCAG 2.2 AA)
8. State-specific content must be jurisdiction-aware
9. Every page needs unique metadata and semantic headings

### NEVER
1. Claim "best lawyer" or imply unbiased recommendation
2. Give state-specific legal advice without counsel review
3. Present tools as legal advice engines
4. Advertise coverage in states not yet cleared
5. Sign contingency agreements unless structure allows it
6. Hardcode content — use JSON CMS
7. Skip disclaimers on any page

### LANGUAGE RULES (CRITICAL)
Use these patterns in ALL user-facing copy:

**DO say:**
- "Based on what you told us, lawyers who typically handle matters like this include..."
- "You may benefit from speaking with a lawyer experienced in..."
- "Availability varies by state and case type."
- "This is educational information, not legal advice."

**NEVER say:**
- "We recommend this lawyer"
- "The best attorney for your case"
- "You should sue" / "You have a case"
- "We guarantee results"
